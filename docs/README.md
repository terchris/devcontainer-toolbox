# Documentation

All documentation for devcontainer-toolbox is built with **Docusaurus** and located in the `website/` folder.

## Documentation Website

**Live site:** https://terchris.github.io/devcontainer-toolbox/

**Source:** `website/docs/`

## Working with Documentation

### View locally

```bash
cd /workspace/website
npm run start -- --host 0.0.0.0
```

### Build for production

```bash
cd /workspace/website
npm run build
```

### Add new pages

1. Create a Markdown file in `website/docs/`
2. Add frontmatter with `title` and `sidebar_position`
3. Restart dev server if adding new folders

See [Contributing → Documentation Website](https://terchris.github.io/devcontainer-toolbox/docs/contributors/website) for the full guide.

## Folder Structure

```
website/
├── docs/                 # All documentation content
│   ├── index.md         # Docs home
│   ├── getting-started.md
│   ├── tools/           # Tool documentation (auto-generated)
│   ├── ai-development/  # AI development guides
│   ├── ai-developer/    # Internal: AI workflow and plans
│   ├── contributors/    # Contributor guides
│   └── ...
├── src/                  # React components and pages
├── static/               # Images, GIFs, favicon
└── docusaurus.config.ts  # Site configuration
```

## Auto-Generated Documentation

Some pages are auto-generated by `dev-docs`:

- `website/docs/tools/index.md` - Tools overview
- `website/docs/tools-details.md` - Tool details
- `website/docs/commands.md` - Command reference

**Do not edit these manually** - they will be overwritten.
